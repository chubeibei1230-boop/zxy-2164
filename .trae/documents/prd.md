## 1. 产品概述

活动手环颜色分组页是一个纯前端管理工具，用于活动开始前整理不同颜色手环、对应批次和发放备注。所有数据保存在浏览器 localStorage 中，无需后端服务，目标用户为活动组织与物资管理人员。

- 解决手环颜色与批次管理混乱、发放状态不透明的问题
- 通过自动检查机制减少人为失误，提升活动物资分装与发放效率

## 2. 核心功能

### 2.1 用户角色

| 角色 | 说明 |
|------|------|
| 活动物资管理员 | 负责手环条目的增删改查、状态流转与批量操作 |

### 2.2 功能模块

1. **手环管理主页**：手环条目的增删改查、筛选、批量状态修改、自动检查提示
2. **批次速览页**：按批次维度重新组织筛选后条目的只读清单视图

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 手环管理主页 | 条目表单 | 新增/编辑手环条目：颜色、批次名、数量、适用人群、优先发放顺序、异常备注、责任人 |
| 手环管理主页 | 条目列表 | 表格展示所有条目，支持单条编辑、删除、状态变更 |
| 手环管理主页 | 筛选栏 | 按颜色、责任人、状态、优先级筛选 |
| 手环管理主页 | 批量操作 | 多选条目后批量修改状态 |
| 手环管理主页 | 检查提示 | 自动检查结果面板，实时显示异常警告 |
| 批次速览页 | 批次分组清单 | 按批次名分组展示筛选后条目，便于快速阅读 |
| 批次速览页 | 批次统计 | 每个批次的数量汇总与状态统计 |

## 3. 核心流程

**主流程**：管理员进入页面 → 新增手环条目 → 设置颜色/批次/数量/优先级等 → 根据需要筛选 → 检查系统自动提示 → 批量修改状态 → 切换批次速览查看全局

**检查流程**：每次数据变动（新增/编辑/删除/批量操作）→ 触发自动检查 → 检查结果实时更新显示

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深灰/炭黑底色 + 手环彩色强调色（用每种手环颜色作为视觉标签）
- 辅助色：琥珀色用于警告、红色用于严重异常、绿色用于正常状态
- 按钮风格：圆角微凸起，hover 带投影变化
- 字体：标题使用 Noto Sans SC Bold，正文使用 Noto Sans SC Regular
- 布局风格：顶部导航 + 侧边筛选栏 + 中央表格/卡片区域
- 图标：使用 lucide-react 图标库

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|----------|
| 手环管理主页 | 顶部导航 | 视图切换按钮、标题、检查结果计数徽章 |
| 手环管理主页 | 筛选栏 | 下拉选择（颜色/责任人/状态/优先级）、重置按钮 |
| 手环管理主页 | 条目表格 | 行选择复选框、颜色色块、状态标签、操作按钮 |
| 手环管理主页 | 条目表单弹窗 | 表单输入、颜色选择器、优先级数字输入 |
| 手环管理主页 | 批量操作栏 | 浮动底部操作栏、状态选择、确认按钮 |
| 手环管理主页 | 检查面板 | 可折叠侧栏/底部面板，异常条目列表，点击定位 |
| 批次速览页 | 批次卡片 | 批次名标题、统计摘要、条目清单 |
| 批次速览页 | 批次统计 | 手环总数、各状态计数、颜色分布 |

### 4.3 响应式

桌面优先设计，最小宽度 1024px；平板（768px-1024px）筛选栏折叠为顶部下拉；移动端暂不做专门适配

### 4.4 3D 场景

不适用
